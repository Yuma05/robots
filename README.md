# プログラミング演習 課題 - Robots

## 概要
robotsは古くからUNIXオペレーティングシステムに付属してきた由緒正しいゲームである。今回の”robots”はそれを簡略化したものである．

プレイヤは悪いロボットと戦っている．プレイヤはロボットからひたすら逃げ続け，ロボット同士がスクラップになるように行動するゲームである．

```
  +------------------------------------------------------------+
	|                                                            |
	|                                                            |
	|                                                            |
	|  +                                                         |
	|             @ *                                  +         |
	|                                                         +  |
	|           +                                                |
	|                                                            |
	|          +                                                 |
	|                                                            |
	|                                                            |
	|                                               +            |
	|                                                            |
	|   +                                                        |
	|                *                                           |
	|                                                            |
	|                                  +                         |
	|                                        +                   |
	|          +                                                 |
	|                                                            |
	+------------------------------------------------------------+
```

上の図で”@”がプレイヤ，”+”がロボットを表す．プレイヤは1回の動作で8方向のいずれかに１マス移動する，全く動かない，ランダムな位置にテレポートするかの選択ができる．

プレイヤが動くと次にロボットが移動する，後出しじゃんけんである．

ロボットは8方向のいずれかに必ず移動する．ロボットは常にプレイヤに最も近づく経路を選択する．

ロボット同士が衝突すると，壊れてスクラップになる．また，ロボットがスクラップに衝突した場合も，そのロボットは壊れる．スクラップは上の図の”*”にあたる．プレイヤがロボットにつかまるとゲームオーバーである．フィールド上のすべてのロボットが壊れると，次のフィールドが現れる．

またプレイヤは，

- フィールドの外
- ロボットのいる場所
- スクラップのある場所

には移動することができない．
